**Общая задумка проекта**  
Платформа для создания и коллективного голосования за законопроекты (или иные предложения). Пользователи делятся на роли:  
- **voter** – только голосует;  
- **proposer** – только создаёт законопроекты;   

Автор публикует законопроект с заголовком, описанием и задаёт период голосования. В течение этого периода (статус `open`) зарегистрированные пользователи отдают свой голос «за» или «против». По окончании голосования статус автоматически меняется на `accepted` или `rejected` (механизм расчёта результатов не описан в структуре, но подразумевается).  

Проект построен на Laravel (наличие таблиц `migrations`, `sessions`, `jobs`, `cache` и др. указывает на это). База данных – MySQL.

---

**Схема базы данных**  

### Таблица `users`  
Хранит учётные записи пользователей.

| Поле                | Тип                        | Описание |
|---------------------|----------------------------|----------|
| `id`                | bigint unsigned, PK, AI    | Уникальный идентификатор |
| `name`              | varchar(255), NOT NULL     | Логин/отображаемое имя |
| `full_name`         | varchar(255), NULL         | Полное имя (опционально) |
| `email`             | varchar(255), UNIQUE, NOT NULL | Электронная почта |
| `github_id`         | varchar(255), UNIQUE, NULL | ID из GitHub (для OAuth) |
| `role`              | enum('voter','proposer','both'), NOT NULL, default 'voter' | Роль пользователя |
| `email_verified_at` | timestamp, NULL            | Дата верификации email |
| `password`          | varchar(255), NOT NULL     | Хеш пароля |
| `remember_token`    | varchar(100), NULL         | Токен для «запомнить меня» |
| `created_at`        | timestamp, NULL            | Дата создания записи |
| `updated_at`        | timestamp, NULL            | Дата обновления |

### Таблица `bills`  
Содержит законопроекты (предложения).

| Поле               | Тип                                                | Описание |
|--------------------|----------------------------------------------------|----------|
| `id`               | bigint unsigned, PK, AI                            | Уникальный идентификатор |
| `user_id`          | bigint unsigned, FK → `users.id`, NOT NULL         | Автор законопроекта |
| `title`            | varchar(255), NOT NULL                             | Заголовок |
| `description`      | text, NOT NULL                                     | Полное описание |
| `status`           | enum('rejected','accepted','standby','open'), NOT NULL, default 'standby' | Текущий статус: `standby` – ожидает начала, `open` – идёт голосование, `accepted`/`rejected` – итог |
| `voting_start_at`  | timestamp, NULL                                    | Дата/время начала голосования |
| `voting_end_at`    | timestamp, NULL                                    | Дата/время окончания голосования |
| `created_at`       | timestamp, NULL                                    | Дата создания |
| `updated_at`       | timestamp, NULL                                    | Дата обновления |

**Связь:** `user_id` → `users.id` (автор).

### Таблица `votes`  
Фиксирует голоса пользователей по конкретным законопроектам.

| Поле         | Тип                               | Описание |
|--------------|-----------------------------------|----------|
| `id`         | bigint unsigned, PK, AI           | Уникальный идентификатор |
| `bill_id`    | bigint unsigned, FK → `bills.id`, NOT NULL | Законопроект, за который голосуют |
| `user_id`    | bigint unsigned, FK → `users.id`, NOT NULL | Пользователь, отдавший голос |
| `vote`       | enum('for','against'), NOT NULL   | Решение: «за» или «против» |
| `created_at` | timestamp, NULL                   | Дата создания |
| `updated_at` | timestamp, NULL                   | Дата обновления |

**Связи:**  
- `bill_id` → `bills.id` (голосование за законопроект);  
- `user_id` → `users.id` (голосующий).  

> **Примечание:** В текущей схеме отсутствует уникальное ограничение на пару (`bill_id`, `user_id`), но логически оно должно быть, чтобы исключить повторное голосование одного пользователя по одному законопроекту. При необходимости это следует добавить миграцией.

---

**Взаимосвязи между таблицами**  
- Один пользователь (`users`) может быть автором многих законопроектов (`bills`) – связь один-ко-многим.  
- Один пользователь может проголосовать за многие законопроекты (`votes`) – связь один-ко-многим.  
- Один законопроект может получить много голосов (`votes`) – связь один-ко-многим.  

Данная структура обеспечивает базовый функционал: создание предложений, управление их статусами и сбор голосов.
